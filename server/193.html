# 自定义任务清单功能

新页面样式

    1. 页面内容（需要从后端获取的参数）
一级元素        二级元素                  三级元素             说明                                                                       原型
任务清单标题
背景图片
底部按钮 按钮图片
按钮交互类型 约定的openType
按钮目标路径 约定的targetPath
按钮交互参数 约定的tabParam
任务清单完成后是否有奖励 用以决定任务清单完成状态下，点击按钮之后的操作：

若【有奖励】，则先弹金币到账的遮罩，2s后自动关闭，再执行配置的按钮交互；

若【没有奖励】，则直接执行配置的按钮交互
任务清单状态 若任务清单状态【进行中】：点击吐司提示“任务清单还未完成哦！”；

任务清单状态【已完成】：点击再判断是否有奖励，执行下一步；

任务清单状态【已领奖】：不显示底部按钮,灰色椭圆提示“已领奖”
任务清单重置机制 用以决定任务重置的机制：
1.用户完成之后不重置；
2.用户完成后立即重置；
3.用户完成后的 0 点重置；
4.无论是否完成，每天 0 点重置
任务清单完成奖励类型 1.金币
2.金额
任务清单完成奖励数值 任务清单完成后奖励的数值
任务 任务排序序号 决定任务排序
任务标题
任务说明
任务状态 进行中
已完成
已领奖
是否有奖励 有奖励：任务已完成状态下，排序序号显示为领奖按钮，点击整个对应任务区域领奖
完成条件 1.指定任务：即需要指定任务id满足条件才算完成
2.随机任务：不指定任务id
样式 1.视频
2.小图无标题
3.大图无标题
4.单列带标题价格
5.纯标题
任务交互对象 任务图片 根据指定的任务id获取的图片,包括的类型：
1.积分墙游戏（速赚）
2.任务赚（高额）
3.免费抽奖奖品
4.梦工厂小游戏

若不指定任务id，则根据任务要求的数量随机取相应的任务图片
任务标题 同上
任务价格 同上，小游戏，免费抽奖奖品为null
交互类型 约定的openType
目标路径 约定的targetPath
交互参数 约定的tabParam
任务类型 用以判断任务的完成方式：
1.积分墙游戏（速赚）：获得奖励即完成；
2.任务赚（高额）：审核通过即完成；
3.观看激励视频广告：看完关闭视频即完成；
4.观看全屏视频广告：跳过视频，看完关闭视频都算完成；
5.参与免费抽奖：参与成功即完成；
6.玩豹趣小游戏：当天体验时长达到要求的时长即完成；
7.玩梦工厂小游戏：当天体验时长达到要求即完成；
8.看小说：当天获得奖励次数达到要求即完成
任务要求完成次数 达到要求完成次数，这个任务才算完成
奖励类型 任务完成后奖励的类型：
1.金币；
2.金额
奖励数量 任务完成后奖励的数值

# 2. 操作栏

  • 未选择记录时：显示筛选条件

常用筛选条件：

  1. 用户id：按uid查询
  2. 用户手机：按user_account查询
  3. 登录：按 is_app_login 查询，下拉选项：（APP：is_app_login=1; h5: is_app_login=0)
  4. 注册日期：按date(registe_time)查询，打开日期范围选择器

高级筛选：

  1. 身份证：按id_card查询
  2. 邀请人：按invitation_uid查询
  3. 银行卡：按user_bank_account查询
  4. 能否中奖：按task_prize_flag查询，下拉选项：（正常：task_prize_flag=0; 禁止中奖：task_prize_flag=1)
  5. 状态：按invalid_flag查询，下拉选项：（正常：invalid_flag=0; 禁止登录：invalid_flag=1）
  6. 应用来源：按channel_id查询，下拉选项：从后端获取选项（channel_user_info.name)
  7. 买号：按buy_account.buy_account查询
  8. 微信昵称：按wx_user_info.wxname查询
  9. 收起：点击收起高级筛选菜单

  1. 收益：收入-支出
