# 1. 邀请高手排行

需要生成假数据组成

假数据要求

  1. 显示前 3 名
  2. 金额数需要会增长，每隔 10 分钟增长随机金额，增长范围在 1~10元之间
  3. 排行榜上的人要变化，但是不频繁变，且一个周期内，不能出现同一个人，之后的金额比之前低的情况，因为金额只会增长。
  4. 一个周期为 每周周日到周一

相关系统参数

  1. 参数名：inviteFakeBasePrize
  2. 说明：用于每周日开始生成假数据时金额的基础金额
  3. 示例：60

# 2. 生成带邀请链接二维码的海报

之前有过相关功能，使用系统参数inviteImgParams配置海报的背景图片和，二维码的邀请链接，但是二维码的位置需要根据UI的新图修改。

相关系统参数

  1. 参数名：inviteImgParams
  2. 说明：用于获取邀请海报的背景图片链接，和邀请链接
  3. 示例：
    [{
 "code": "1",
 "inviteBgImg": "static/mobile/images/invite/invite_bg_01.png",
 "qaSize": 375,
 "inviteUrl": "http://www.gszhuan.com/page/channel_invite#/shw_generalizePage?ic=%s",
 "topHeight": 1282
 }]
  4. inviteBgImg 是图片练级；inviteUrl 是邀请链接，用于生成二维码，生成二维码之前，需要用户的邀请信息（如邀请码，uid等）替换到链接中的 %s
  5. 备注：生成二维码的邀请链接即做邀请百元活动时分享出去的邀请链接

# 4. 邀请等级区分

等级要求

用户邀请等级 要求 奖励
银牌团长 邀请活跃人数>10人 1. 徒弟获得游戏赚回调 / 任务赚奖励时，奖励师傅（徒弟所得金币的10%）；
2. 徒孙获得游戏赚回调 / 任务赚奖励时，奖励师傅（徒弟所得金币的5%）
金牌团长 邀请活跃人数>50人 1. 徒弟获得游戏赚回调 / 任务赚奖励时，奖励师傅（徒弟所得金币的13%）；
2. 徒孙获得游戏赚回调 / 任务赚奖励时，奖励师傅（徒弟所得金币的7%）
钻石团长 一级邀请人数>1000人，一二级邀请人数>8000人，一二级邀请活跃人数占比 > 30% 1. 徒弟获得游戏赚回调 / 任务赚奖励时，奖励师傅（徒弟所得金币的13%）；
2. 徒孙获得游戏赚回调 / 任务赚奖励时，奖励师傅（徒弟所得金币的7%）

名词说明

  1. 邀请活跃：指该用户邀请的，完成过 2 次或以上 每日任务 的人
  2. 一级邀请：指该用户邀请的人（徒弟）
  3. 二级邀请：指该用户邀请的人邀请的人（徒孙）
  4. 游戏赚回调：指获得多游/享玩 游戏奖励回调
  5. 任务赚奖励：指管理后台/移动端任务后台 审核通过任务

团长升级的触发点

  1. 用户注册，导致邀请人数达到要求
  2. 用户完成每日任务，导致邀请活跃达到要求

相关系统参数 1 配置徒弟徒孙做任务，团长的分红奖励比例

  1. 参数名：inviteLeaderBonusPrize
  2. 说明：徒弟徒孙做游戏/任务赚后，团长获得的奖励分红比例
  3. 示例：
    {
"silver":{
    "disciple":10,
    "subDisciple":5
},
"gold":{
    "disciple":13,
    "subDisciple":7
},
"diamond":{
    "disciple":13,
    "subDisciple":7
}
}
  4. 参数说明：
参数 子参数 类型 说明
silver string 银牌团长
gold string 金牌团长
diamond string 钻石团长
disciple int 徒弟完成游戏/任务赚 时，按这个比例乘以任务奖励，分红给团长。单位：%，所以使用时需要先除以100
subDisciple int 徒孙完成游戏/任务赚 时，按这个比例乘以任务奖励，分红给团长。单位：%，所以使用时需要先除以100

相关系统参数 2 配置每级团长的要求

  1. 参数名：inviteLeaderIdCondition
  2. 说明：升级团长的要求
  3. 示例：
    {
"silver":{
    "activeNum":10
},
"gold":{
    "activeNum":50
},
"diamond":{
    "disciple":1000,
    "discipleAndSubDisciple":8000,
    "oneAndTwoActiveRate"30
}
}
  4. 参数说明：
参数 子参数 类型 说明
silver string 银牌团长
gold string 金牌团长
activeNum int 邀请活跃人数，完成过 2 次每日任务的用户为活跃用户，每日任务为activity_user_list表中list_type = '00' 的记录
diamond string 钻石团长
disciple int 徒弟人数
discipleAndSubDisciple int 徒弟和徒孙的总人数
oneAndTwoActiveRate int 徒弟和徒孙中的活跃用户占比。单位:%

# 5. 分红钻石功能

发放分红钻石的逻辑

  1. 团长升级时，可拆红包，红包拆出的内容如下：
红包内容 几率 奖励数值范围 说明
金币 1 100~200 从范围内随机取金币奖励到用户金币账户中，增加金币流水明细，备注：邀请身份升级拆红包奖励
分红钻石 0 1 给当前用户记录获得 1 枚钻石

WARNING

  1. 几率，奖励数值范围可通过系统参数配置
  2. 升级为钻石团长，拆红包必得 分红钻石
  1. 用户每获得20 次 游戏赚/任务赚 奖励，可拆红包，红包拆出的内容如下：
红包内容 几率 奖励数值范围 说明
金币 1 100~200 从范围内随机取金币奖励到用户金币账户中，增加金币流水明细，备注：累计完成 20/40/80/... 次 游戏/任务 赚拆红包奖励
分红钻石 0 1 给当前用户记录获得 1 枚钻石

WARNING

  1. 几率，奖励数值范围可通过系统参数配置
  2. 累计完成 1000 个游戏/任务 赚，拆红包必得 分红钻石

通知客户端可拆红包

当用户有上述红包可拆时，需要在m/app-home/ v5 接口中 通过新增参数 isBonusPop 告诉客户端需要弹窗拆红包，1 表示需要弹窗，0 表示不用弹窗拆红包。

分红钻石的作用

每隔 一段间隔时间 用 分红总额 按已发放的钻石总数平分，再按用户拥有的钻石数发放对应金币奖励

WARNING

  1. 间隔时间从系统参数配置，单位：分钟
  2. 分红总额从系统参数配置，单位：金币

相关系统参数

  1. 参数名：bonusDiamondInfo
  2. 说明：分红钻石的相关参数
  3. 示例： { "getDiamond":{ "minPrize":10, "maxPrize":200, "diamondRateUpgrade":0, "diamondRateTask":0 }, "diamondPrize":{ "prizeInterval":10, "totalPrize":20 } }
  4. 参数说明
参数 子参数 类型 说明
getDiamond string 获取钻石的相关参数
minPrize int 打开红包时抽中金币数的最小值
maxPrize int 打开红包时抽中金币数的最大值
diamondRateUpgrade int 升级团长时，打开红包获得分红钻石的几率，单位：%
diamondRateTask int 完成指定数量任务时，打开红包获得分红钻石的几率，单位：%
diamondPrize string 分红钻石分红的相关参数
prizeInterval int 触发分红的时间间隔，单位：分钟，如10，则指每10分钟分红一次
totalPrize int 每次分红的总金币，单位：金币，分红到用户的金币账户

WARNING

用户打开红包，要么获得金币，要么获得钻石，不能两者兼得
