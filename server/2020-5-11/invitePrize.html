# 邀请奖励-任务清单奖励

需求类型：

  • 新增

新增系统参数：

  • 参数名：invitePrizeTaskList

  • 数据示例：

    {
  "1":{
      "account":"02",
      "prizeNum":"150",
      "remark":"邀请 %s 首次完成每日任务"
  },
  "2":{
      "account":"02",
      "prizeNum":"100",
      "remark":"邀请 %s 第 2 次完成每日任务"
  },
  "3":{
      "account":"02",
      "prizeNum":"50",
      "remark":"邀请 %s 第 3 次完成每日任务"
  }
}

  • 参数说明

    参数 类型 说明
    account string 奖励账户：
    01表示奖励金额到free_money；
    02表示奖励金币到coins_num
    prizeNum string 奖励数值
    remark string 增加明细记录时的备注

流程：

请求URL：

  • /my/act-bottom-btn/action

请求参数：

参数名 类型 说明
code string 任务清单标识
userListId string 用户参与任务清单记录的标识

流程说明：

流程 操作说明
被邀请人 当前完成了任务清单的用户
邀请人uid 根据uid 查询 sys_user 表中的 invitation_uid
已完成任务清单数 根据uid 统计 activity_user_list 表中 status = '1',list_type = '00' 的记录数
查询任务清单邀请奖励参数 根据已完成任务清单数作为key，查询系统参数invitePrizeTaskList，获取对应的value
奖励邀请人 1. 新增invite_prize记录：
    prize_method为 上一步查到的value中的remark(用被邀请人手机号代替remark中的%s)，
    prize_money为 上一步查到的value中的prizeNum/100,
    uid为邀请人uid，
    prize_time 为当前时间，
    invalid_flag 为 0，
    task_id 为客户端传的userListId，
    invite_id 为 被邀请人uid，
    prize_type为 4019

2. 给邀请人新增金币到账记录:
    direction 为 大写C，
    tradetype 为 2020，
    coinAmt 为 上一步查到的value中的prizeNum，
    balance为奖励后的新金币余额，
    remark为上一步查到的value中的remark(用被邀请人手机号代替remark中的%s)，
    reference_id为 第一点新增的invite_prize记录id

3. 更新邀请人的金额余额（sys_user.coins_num）为奖励后的余额
