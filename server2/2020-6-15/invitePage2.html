# 阶段数据

参数 类型 说明
inviteBalance double 邀请账户余额
withdrawStage int 当前可提现的阶段，判断方法见上文【提现】
withdrawAmt double 当前可提现的阶段目标金额，根据当前可提现的阶段对应的target目标金额
curStage int 当前用户所处的阶段
curStageTarget double 当前用户所处阶段的目标金额
curStageBalance double 用户在当前阶段已获得的邀请奖励
curStageDesc string 第一、二、三阶段返回：再邀 n 人完成每日任务可提 m 元（n为距离目标还差多少人完成每日任务，m为当前阶段的目标金额）

其它阶段返回null

# 联系方式（名片）

需要有保存联系方式的接口，和获取联系方式的接口

参数 类型 说明
phone string 手机号
wxCode string 微信号
QQ string QQ号

WARNING

名片可设置是否对师傅公开，是否对徒弟公开，只有当设置公开时，对应身份的人请求时才能返回信息；否则返回：对方未公开名片
若设置了公开，但是没有填写任意三项之一信息，则返回：对方未保存任何联系方式

# 邀请的好友数据

参数 子参数 类型 说明
summary array 总概数据
firstInviteActiveNum int 一级邀请有完成过每日任务的人数
firstInviteNum int 一级邀请总人数
firstInvitePrize double 一级邀请带来总邀请收益，包括第一、二阶段的奖励，以及其它阶段的徒弟完成任务分红
secondInviteActiveNum int 二级邀请有完成过每日任务的人数
secondInviteNum int 二级邀请总人数
secondInvitePrize double 二级邀请带来总邀请收益，包括第三、四、五阶段二级邀请者完成任务分红
totalInviteActiveNum int 邀请的总人数中有完成过每日任务的人数，包括 一级，二级，三级 ... 无限级
totalInviteNum int 邀请的总人数，包括 一级，二级，三级 ... 无限级
totalInvitePrize double 邀请的总收益
firstInviteList array 徒弟数据
wxName string 徒弟微信昵称
isActive int 徒弟是否完成过每日任务，1 标识完成过，即活跃；0标识每完成过
inviteNum int 徒弟邀请的一级邀请人数

WARNING

徒弟数据列表需要根据客户端传的排序方式排序返回：

  1. 默认按注册日期从近到远排序；
  2. 按徒弟邀请的一级邀请人数从多到少排序；
  3. 按徒弟邀请的一级邀请人数从少到多排序；
