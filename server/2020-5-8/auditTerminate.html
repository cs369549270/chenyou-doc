# 任务审核终止

需求类型：

  • 梳理

流程：

请求URL：

  • 管理后台任务审核接口：/app-task/user-attend/audit
  • 移动端任务管理后台审核接口：/my/seller-app-task/audit

请求参数：

参数名 子参数 类型 说明
data json string 存在多条记录批量处理的情况
id int 用户参与记录的标识,对应user_attend_task_line表id
appid int 任务标识
status string 任务状态，终止传'06'
remark string 驳回原因
phone string 用户手机号
uid int 用户标识
taskid string appid对应little_app_task_info 的id

流程说明：

流程 操作说明
终止任务 1. 根据appid，uid 查询 user_attend_app_line 表中 status = '01' 的记录，修改status 为 '06'，endOn 为 当前时间
2. 根据appid，uid 查询 user_attend_task_line 表中 status = '01' 的记录，修改status 为 '06'，reject_reason 为 前端传的remark
任务状态 根据appid 查询 app_task_info 表中的 status
可领取数量 1. 根据appid 查询 app_task_plan 表中的 start_time <= now() 的 release_num 总和 → 已投放份数
2. 根据appid 查询user_attend_app_line表中status = '01' 的任务记录数 → 进行中份数
3. 已投放份数 - 进行中份数 → 可领取数量
剩余时间 1. 根据appid 查询 app_task_info 表中的downdt → 截止时间
2. 截止时间 - now() → 剩余时间
发布来源 根据appid 查询 app_task_info 表中的ori_type ： 01 表示 运营发布 ； 02 表示 渠道发布
退款 1. 根据 appid 查询 app_task_info 中的渠道标识 ruid ,title,seller_pay_price

2. 在 fn_io_record 中添加渠道的退款记录:
    comment = '任务【 title 】审核终止，任务已过期，退款'，
    trade_type = '3072' ，
    money = seller_pay_price/100

3. 增加渠道的余额（sys_user 中的 free_money) + seller_pay_price/100
进行中数量 根据appid 查询user_attend_app_line表中status = '01' 的任务记录数
修改任务状态 为 1 根据appid 修改 app_task_info 表中的status 为 1
修改任务状态 为 7 根据appid 修改 app_task_info 表中的status 为 7
