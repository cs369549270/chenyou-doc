# 任务结算

需求类型：

  • 优化

流程：

请求URL：

  • /my/seller-app-task/charrenOperation

请求参数：

参数名 类型 说明
id string 操作标识，结算传"js"
taskid int 任务标识

流程说明：

流程 操作说明
可领取数量 1. 根据appid 查询 app_task_plan 表中的 start_time <= now() 的 release_num 总和 → 已投放份数
2. 根据appid 查询user_attend_app_line表中status = '01' 的任务记录数 → 进行中份数
3. 已投放份数 - 进行中份数 → 可领取数量
修改任务状态 为 7 根据appid 修改 app_task_info 表中的status 为 7
结算 1. 根据taskid作为id 修改 app_task_info 中的downdt 为当前时间

**2. 根据 taskid作为id 查询 app_task_info 中的渠道标识 ruid ,title,seller_pay_price **

3. 在 fn_io_record 中添加渠道的退款记录:
    comment = '任务【 title 】结算'，
    trade_type = '3065' ，
    money = （seller_pay_price/100 * 可领取份数），
    free_money 为（sys_user 中的 free_money) + （seller_pay_price/100 * * 可领取份数）

4. 更新结算后的用户余额 sys_user.free_money
进行中数量 根据appid 查询user_attend_app_line表中status = '01' 的任务记录数
修改任务状态 为 0 根据appid 修改 app_task_info 表中的status 为 0
