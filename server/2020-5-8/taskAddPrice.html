# 任务加价

需求类型：

  • 优化

流程：

请求URL：

  • /my/seller-app-task/charrenOperation

请求参数：

参数名 类型 说明
id string 操作标识，加量传"jj"
taskid int 任务标识
price string 每份添加的价格，单位：元

流程说明：

流程 操作说明
任务状态 根据appid 查询 app_task_info 表中的 status
409 任务已被领取完，已下架 状态码返回409，提示：“任务已被领取完，已下架”
余额 根据 uid 查询 sys_user 表中的free_money
扣款金额 1. 根据taskid 作为id 查询app_task_info 表中的release_num → 总份数
2. 根据taskid 作为appid 查询 user_attend_task_line 表中 status !='06' 的记录数 → 已领取数量
3. 前端传的price * （总份数 - 已领取数量） → 扣款金额
409 余额不足 状态码返回409，提示：“余额不足”
扣款 1. 新增用户资金明细记录:
    money_type 为 2，
    money 为 扣款金额，
    comment修改为：“任务【app_task_info.title】加价”，
    reference_id 为 前端传的taskid，
    free_money 为 扣款后的新余额，
    trade_type 为 3067

** 2. 更新用户扣款后的余额**
加价 1. 根据用户uid查询apptask_channel_user 表中的task_rate → 放佣比例
2. 根据taskid 作为 id 查询更新app_task_info.seller_pay_price 为 （原数值 + 前端传的 price100）,coin 为 （更新后的seller_pay_price * 放佣比例task_rate）向上取整
3. 根据taskid 作为 appid 查询更新little_app_task_info.seller_pay_price 为 （原数值 + 前端传的 price100）,coin 为 （更新后的seller_pay_price * 放佣比例task_rate）向上取整
