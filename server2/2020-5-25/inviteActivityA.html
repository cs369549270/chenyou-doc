# 邀请活动A - 一次拆一个红包

# 流程

流程图

流程说明

流程 说明
红包开启金额 用户开启一个新红包时，红包的金额，从系统参数配置的闭区间范围中随机获取，保留 2 位小数
新用户手机绑定微信 指绑定手机的操作，（包括app绑定手机、app闪验一键登录、h5绑定手机）
新用户填写邀请码 指在新页面【填写邀请码】提交邀请码成功
给用户增加红包金额 增加已开启的红包金额，增加的份额计算方式见下文
用户提现 用户在邀请活动页提现红包金额
开启时限 用户从红包开启，到达到提现门槛的时限，系统参数配置
提现时限 用户从达到提现门槛，到提现成功的时限，系统参数配置
微信打款给用户 微信付款金额到用户微信零钱，需要在提现记录表添加一条提现成功的记录

# 增加份额计算方法

流程

流程说明

流程 说明
剩余可增份额 提现门槛 - 当前红包金额
剩余需邀请人数 需邀请总人数 - 开启当前红包后已邀请注册登录人数

需邀请总人数：系统参数配置

开启当前红包后已邀请注册登录人数：用户邀请的，注册时间在当前红包开启时间之后，且已绑定微信的用户数
相关系数 系统参数配置
可增最大额 取得最大额后，给用户增加的份额在 闭区间 [0,可增最大额]中随机获取一个 2 位小数的数值，最小值 0.01

# 相关系统参数

内容示例

{
    "minAmountOpen":"93",
    "maxAmountOpen":"96",
    "openTime":86400,
    "withdrawTime":259200,
    "withdrawLimit":"100",
    "totalInvite":60,
    "coefficient":"0.06",
    "share":{
        "link":"",
        "title":"",
        "desc":"",
        "icon":"",
        "content":""
    }
}

参数说明

参数 类型 说明
minAmountOpen string 开启红包时红包金额的最小值
maxAmountOpen string 开启红包时红包金额的最大值
openTime int 红包开启，到达到提现门槛 的时限，超时则关闭红包，单位：秒
withdrawTime int 红包达到提现门槛，到提现成功 的时限，超时则关闭红包，单位：秒
withdrawLimit string 提现门槛
totalInvite int 需邀请总人数，要达到提现门槛需要用户邀请的人数
coefficient string 相关系数，用于计算每次邀请增加的红包金额
link string 邀请链接，接口返回时需要用用户的邀请码替换内容中的%s
title string 分享的标题
desc string 分享的描述
icon string 分享的图标链接
content string 分享的文案，接口返回是需要用用户的邀请码替换内容中的%s
