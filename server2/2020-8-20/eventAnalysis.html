# 名词定义

为了方便之后更多的分析工具，定义几个统一的名词

名词 说明
事件 用户完成了某个操作，称为 1 次事件。比如注册，登录，获得1次游戏赚奖励，完成1次任务清单，拆1个红包等等
事件属性 事件本身所拥有的属性，比如注册的属性包括：注册方式（验证码注册，闪验注册），注册的时间等，一个事件对应多个属性
事件指标 事件分析的目标数据，比如注册的总人数，增长率等，一个事件对应多个指标
公共属性 跟用户本身相关的属性，可适用于所有事件。比如用户渠道来源，用户设备号，余额，已赚收益等

# 根据时间查询分组数据

# sql示例：

select a.date_str,ifnull(t1.eventUids,0) as eventUids
from (select date_format(date_str,'%Y-%m-%d %k:%i')as date_str # 时间粒度
      from (
               SELECT @i := date_add(@i, interval - 1 minute ) as date_str
               FROM (SELECT @i := date_add('2020-08-21 23:00:00', interval + 1 minute) from sys_user) t1) t2 # 这里date_add的时间要取endDate
      where
        t2.date_str between '2020-08-21 00:00:00' and '2020-08-21 23:00:00' # 时间范围
     ) a
left join
     (select date_format(create_time,'%Y-%m-%d %k:%i')as date, count(distinct(uid))as eventUids # 时间粒度
        from user_event_record
     where event_id = 1 # 事件指标对应的sql
       and property_1 = '' # 事件属性
       and create_time between '2020-08-21 00:00:00' and '2020-08-21 23:00:00' # 时间范围
       and uid in (select uid from sys_user where yxz_income = '') # 公共属性
     group by date_format(create_time,'%Y-%m-%d %k:%i') # 时间粒度
         ) t1
     on a.date_str = t1.date

# 需要基础指标运算的情况：

  1. 根据需求的指标，到event_indicator_info 表中查询，若 calculation = 0 说明是有其它指标运算得出的，则根据 calculation_op 运算公式中的变量名 找到 对应指标的计算方式，以此类推，直到找到运算的指标 calculation = 1 （即都可以通过sql查询）
  2. 将所有sql 拼上时间范围，时间粒度，公共属性，若sql 中的event_id 与 请求的event_id 相同，还需要拼上事件属性，若不同则不需要。
  3. 将所有sql 通过 left join 联表查询
  4. 在最外层的select 中 根据 calculation_op 的运算公式运算

sql示例：示例中第二个指标的对应的 event_id = 2 ，假设请求的是 event_id = 1 ，所以不加事件属性的参数

select a.date_str,ifnull(t1.eventUids + t2.eventUids2,0) as eventUids
from (select concat(year(date_str),floor((date_format(date_str,'%m')+2)/3))as date_str # 时间粒度
      from (
               SELECT @i := date_add(@i, interval - 3 month ) as date_str
               FROM (SELECT @i := date_add('2020-08-21 23:00:00', interval + 3 month) from fn_io_record) t1) t2 # 这里date_add的时间要取endDate
      where
        t2.date_str between '2010-01-10 00:00:00' and '2020-08-21 23:00:00' # 时间范围条件
     ) a
left join
     (select concat(year(create_time),floor((date_format(create_time,'%m')+2)/3))as date, count(distinct(uid))as eventUids # 时间粒度
     from user_event_record
     where event_id = 1 # 事件id ，事件指标对应的sql，若是由其它指标运算得出的，这里先联表查出基础指标，再在最外层的select中运算
       and property_1 = '' # 事件属性
     and create_time between '2010-01-10 00:00:00' and '2020-08-21 23:00:00' # 时间范围
     and uid in (select uid from sys_user where yxz_income = '') # 公共属性
         group by concat(year(create_time),floor((date_format(create_time,'%m')+2)/3)) # 时间粒度
         ) t1
     on a.date_str = t1.date
left join
     (select concat(year(create_time),floor((date_format(create_time,'%m')+2)/3))as date, count(distinct(uid))as eventUids2 # 时间粒度
     from user_event_record
     where event_id = 2 # 事件id ，事件指标对应的sql，若是由其它指标运算得出的，这里先联表查出基础指标，再在最外层的select中运算
     and create_time between '2010-01-10 00:00:00' and '2020-08-21 23:00:00' # 时间范围
     and uid in (select uid from sys_user where yxz_income = '') # 公共属性
         group by concat(year(create_time),floor((date_format(create_time,'%m')+2)/3)) # 时间粒度
         ) t2
     on a.date_str = t2.date

# 统计总数和平均数

sql示例：

select t1.eventUids as eventUids, # 总数直接查出来
    round(t1.eventUids / (timestampdiff(day,'2010-01-10 00:00:00','2020-08-21 23:00:00')+1),2)as average # timestampdiff函数中使用的参数依次是：时间粒度，开始时间，结束时间
from
     (select count(distinct(uid))as eventUids
     from user_event_record
     where event_id = 1 # 事件id ，事件指标对应的sql，若是由其它指标运算得出的，这里先联表查出基础指标，再在最外层的select中运算
       and property_1 = '' # 事件属性
     and create_time between '2010-01-10 00:00:00' and '2020-08-21 23:00:00' # 时间范围
     and uid in (select uid from sys_user where yxz_income = '') # 公共属性
         ) t1

  • 查询前判断下时间参数是否合理：
  1. 时间粒度 = day：
    1. 需要符合的条件：结束时间 - 开始时间 < 60 天
    2. 否则返回错误信息：“以天为时间粒度，范围不可超过 60 天”
  2. 时间粒度 = week：
    1. 需要符合的条件：结束时间 - 开始时间 < 366 天
    2. 否则返回错误信息：“以周为时间粒度，范围不可超过 366 天”
  3. 时间粒度 = month：
    1. 需要符合的条件：结束时间 - 开始时间 < 1000 天
    2. 否则返回错误信息：“以周为时间粒度，范围不可超过 1000 天”
  4. 时间粒度 = quarter：
    1. 需要符合的条件：结束时间 - 开始时间 < 1000 天
    2. 否则返回错误信息：“以季为时间粒度，范围不可超过 1000 天”
  5. 时间粒度 = year：
    1. 需要符合的条件：结束时间 - 开始时间 < 10 年
    2. 否则返回错误信息：“以年为时间粒度，范围不可超过 10 年”
  6. 时间粒度 = minute：
    1. 需要符合的条件：结束时间 - 开始时间 < 60 分钟
    2. 否则返回错误信息：“以周为时间粒度，范围不可超过 60分钟”
  7. 时间粒度 = hour：
    1. 需要符合的条件：结束时间 - 开始时间 < 24 小时
    2. 否则返回错误信息：“以周为时间粒度，范围不可超过 24 小时”

# 校验自定义指标的sql

  1. 判断是否为 select 开头，只能接收select 语句
  2. 判断是否包含 as ，必须包含 as 重命名变量，且重命名的变量名必须与传过来的指标变量名相同
  3. 检查 from 和 where 中的表名是不是 user_event_record，若不是，返回：表名错误
  4. 试运行sql，看数据库是否报错，若报错将错误信息返回
  5. 运行时长不得超过 3 s ，否则返回：sql运行超时，无法提交
